## 总体判断

你的 B 站 AI 收藏不是泛泛的“AI 新闻”或“模型测评”，而是明显偏实操：

- 第一类是 Codex / Agent 工具链：怎么装、怎么选插件、怎么接第三方 API、怎么把 Agent 用到网页、动画、PPT、表格、浏览器自动化等任务。

- 第三类是权益/账号类：Google AI Pro 订阅获取流程，偏资源获取，不是技术学习本身。
- 第四类是 AI 时代的工程学习路线：面向机电/机械学生，关注未来工程师该补哪些跨栈能力。

## 后续如果要做“真正逐字内容总结”

需要补齐其中一个转写路径：

1. 配置 `summarize transcriber setup` 里的 whisper.cpp 或 ONNX parakeet/canary。
2. 用 B 站视频流 API 下载音频，再转写成文字。
3. 把转写稿输入 `summarize - --lang zh --length long --plain`。
4. 对 Codex 操作演示类视频，再抽关键画面 OCR，补足屏幕上的配置项和命令。

目前我已经验证：

- `@steipete/summarize` CLI 可用，版本为 `0.18.1`。
- `summarize --extract` 对 B 站 URL 只能抽网页 HTML，无法直接拿到视频内容。
- B 站 `playurl` API 可以拿到音频流地址，后续可走“下载音频 → 本地转写 → summarize”的路线。
- 本机已有 `ffmpeg` 和 `yt-dlp`，但没有可用本地语音转写器。
